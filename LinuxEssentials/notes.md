## Modulo 2 - Operating Systems

$\color{blue}{\text{**Embedded Systems**}}$ means that the system is designet to do a specific task on hardware opmitized for onyl that purpose.

In Lunix, RPM can be defined as $\color{blue}{\text{**The Package Manager**}}$ program for Red Hat Linux. 

A long software $\color{blue}{\text{**release cycle**}}$ is: Valued by businesses that want stability.

Bundling utilities, managment tools, and application software with a Linux kernel is called a $\color{blue}{\text{**Distribution of Linux**}}$

A software release cycle describes $\color{blue}{\text{**how often upgrades come out for software**}}$

If a software release is in a state in that it has many new features that have not been rigorously tested, it is typically referred to as $\color{blue}{\text{**beta software**}}$

Apple's OS X is a $\color{blue}{\text{**fully certified UNIX distribution**}}$

Microsoft Windows has a scripting environment called $\color{blue}{\text{**PowerShell**}}$, offers both $\color{blue}{\text{**desktop and server products**}}$ and is generally $\color{blue}{\text{**backwards compatible**}}$ with previous versions

When choosing a distribution of Linux, you should consider:  
* Will users require a GUI
* Does the distribution offer a "stable" version
* Does your organization require long-term support for the system
* If the application software is supported by the distribution
* Will commercial support be required for the OS
